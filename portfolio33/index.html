웹표준 사이트를 만들면서 배웠던 점

  • 웹 표준의 개념을 이해하다

    사이트를 만들며 웹 표준의 개념을 공부하였습니다. 웹 표준은 '웹에서 표준적으로 사용되는 기술이나 규칙'으로 W3C에서 나온 권고안을 말하며, 웹 표준을 지키기 위해선 웹 접근성과 웹 호환성을 필수적으로 지켜야 합니다. 웹 접근성은 모든 사용자가 환경, 지식의 여부, 장애의 여부 등에 상관없이 동등하게 웹을 사용할 수 있어야 한다는 개념이며, 웹 호환성은 어떤 브라우저에서도 웹을 사용할 수 있어야 한다는 개념입니다.

  • 이미지 대체 텍스트란?

    기존에 막연하게 알았던 '이미지 대체 테스트'의 개념을 명확하게 알 수 있었습니다. '이미지 대체 텍스트'란 웹 접근성을 준수하기 위한 한 방법으로 사용자가 이미지를 볼 수 없을 때, 이미지를 대신 설명해주는 텍스트입니다. 의미있는 이미지의 경우, img 태그와 함께 alt 속성을 필수적으로 사용해 웹 접근성을 준수하지만, background의 경우 alt 속성을 사용할 수 없으므로 IR 기법을 사용합니다.

  • IR 기법이란?

    IR기법은 CSS로 이미지 대체 텍스트를 제공하는 기법입니다. 이전까진 background에도 대체 텍스트를 사용할 수 있다는 사실을 알지 못하였으나, 이 사이트를 만들며 IR 기법에 대해 처음 알게 되었고 웹 접근성을 더욱 더 알아가는 계기가 되었습니다.

  • 제이쿼리의 사용을 이해하다

    제이쿼리로 탭메뉴를 만들고 간단한 메서드를 활용하면서 간단한 동적 요소들을 만들 수 있게 되었습니다. 특히 $와 메서드 체인을 이용해 자바스크립트보다 간편하게 사용가능한 제이쿼리의 장점을 이해하게 되었습니다.

  • 팝업과 모달

    제이쿼리로 팝업창과 모달창을 만들면서 둘의 차이점을 알게 되었습니다. 특히 window.open()을 처음 사용하면서 새창에서 여는 팝업의 개념을 이해하게 되었습니다.

  • 제이쿼리 플러그인을 통한 다양한 표현

    Slick.js, lightgallery.js 등 제이쿼리 플러그인을 사용하는 방법을 배웠습니다. 매인배너와 갤러리 등을 만들고 이를 재생, 정지 등 조작하여 콘텐츠를 보다 다양하게 표현하는 방법을 배웠습니다.

  • 미디어쿼리를 익히다.

    반응형에서 필수적인 미디어쿼리를 사용하며 관련 내용을 배웠습니다. 미디어쿼리는 CSS에서 특정 화면 너비나 해상도에서 특정 스타일을 사용하고 싶을 때 사용합니다. 본 사이트에선 max-width가 768px 일 때, 모바일 화면으로 전화시켰고 600px, 480px, 320px 일 때 요소들을 추가적으로 조정해 깨짐없이 구현하도록 했습니다.

  • 해상도에 맞춘 이미지 변환을 배우다

    이 사이트를 만들면서 반응형에서 특정 해상도일 때, 이미지 해상도를 맞추는 것을 처음 접하였습니다. 미디어쿼리를 사용해 화면 비율이 1.5일 때 이미지를 고해상도로 변화시켰고, img 태그에는 srcset 속성을 사용해주었습니다.

  • Fontawesome을 이용한 아이콘 표현

    아이콘 이미지를 삽입할 때 이미지 파일이 아닌 FontAwesome의 Icon을 연결하여 사용해 보았습니다. 형식은 웹폰트이나 안에는 글자가 아닌 아이콘 이미지가 들어있는 형태로, css를 사용해 색 변형을 줄 수 있어 직접 이미지파일을 사용하는 것보다 더 편리하고 깔끔하게 아이콘을 표현할 수 있었습니다.

    Slick.js, lightgallery.js 등 제이쿼리 플러그인을 사용을 통해 매인배너와 갤러리 등을 만들고 이를 재생, 정지 등 조작하여 콘텐츠를 보다 다양하게 표현하는 방법을 배웠습니다.
